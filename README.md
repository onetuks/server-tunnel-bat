# 🚀 Multi-Environment SSH Tunneling Automation

이 도구는 개발(Dev), 품질(QA), 운영(Prod) 등 다양한 인프라 환경의 SSH 터널링을 설정 파일을 기반으로 한 번에 자동화하여 열어주는 도구입니다. Jump Host(베스천) 경유 여부와 다중 포트 포워딩을 지원합니다.

---

## 📂 파일 구조

```text
ce-tunnel/
├── run_tunnel.bat           # 1. 터널링 실행 (더블클릭)
├── stop_tunnel.bat          # 2. 터널링 종료 (더블클릭)
├── tunnel_config.json       # 3. 로컬 환경 설정 파일 (비밀번호 및 호스트 정보)
├── tunnel_sendkeys.ps1      # 4. 자동 패스워드 입력 파워쉘 스크립트
└── README.md                # 5. 본 문서
```

## 🛠️ 1. 사전 설정 (`tunnel_config.json`)

최초 실행 전, `tunnel_config.template.json` 파일을 복사하여 **`tunnel_config.json`** 파일을 생성하고 본인의 접속 정보를 설정해야 합니다. (실제 비밀번호가 포함된 `tunnel_config.json`은 `.gitignore`에 등록되어 있어 Git에 업로드되지 않습니다.)

### ⚙️ 설정 프로퍼티 설명

| 프로퍼티 | 설명 |
| :--- | :--- |
| **`Env`** | 환경 식별 키 (`dev`, `qa`, `prod` 등) |
| **`EnvName`** | 콘솔 창 제목 및 터널 이름에 매핑될 화면 출력용 이름 |
| **`Enabled`** | 해당 터널링의 활성화 여부 (`true` / `false`) <br> * 여러 개를 `true`로 설정하면 한 번에 여러 개의 터널이 동시에 생성됩니다. |
| **`RemoteHost`** | 최종 원격 대상 서버의 SSH 접속 주소 (`user@ip`) |
| **`RemotePassword`**| 최종 원격 대상 서버의 로그인 비밀번호 |
| **`JumpHost`** | (선택사항) 경유할 베스천(점프) 호스트의 주소 (`user@ip`) <br> * 빈 문자열(`""`)로 둘 경우 직접 연결(Direct Connect)로 실행됩니다. |
| **`JumpPassword`** | (선택사항) 베스천 호스트의 로그인 비밀번호 |
| **`Ports`** | 터널링 포워딩할 로컬 포트 목록 (배열 형식) <br> * 예: `["8090", "8082", "3306"]` 등 개수 제한 없이 설정 가능합니다. |

> [!WARNING]
> 여러 환경을 동시에 활성화(`"Enabled": true`)하는 경우, 포트가 서로 충돌하지 않도록 각 환경의 `Ports` 설정 번호가 중복되지 않게 지정하셔야 합니다.

## ⚡ 2. 사용 방법

### **1단계: 터널링 실행**
- **`run_tunnel.bat`** 파일을 마우스로 더블클릭하여 실행합니다.

### **2단계: 자동 로그인 완료 대기**
- `tunnel_config.json`에서 `Enabled: true`인 각 환경의 터널링 창(Classic Console)이 차례대로 열립니다.
- **🚨 주의**: 터널링 창이 활성화되고 패스워드가 자동으로 끝까지 기입될 때까지 마우스 클릭이나 키보드 입력을 하지 마십시오. (마우스 클릭 등으로 창 포커스가 다른 곳으로 이동하면 패스워드 입력 자동화가 실패할 수 있습니다.)

### **3단계: 연결 상태 확인**
- 실행된 개별 터널링 콘솔 창에 최종 원격 서버의 쉘 프롬프트(예: `[user@hostname ~]$`)가 떠 있는지 확인합니다.
- 웹 브라우저를 열고 `http://localhost:[설정한포트]` (예: `http://localhost:8090`) 주소로 접근하여 목적지 어드민 웹페이지나 API가 잘 호출되는지 확인합니다.

---

## 🛑 3. 터널링 종료

- **특정 터널링만 개별 종료**: 해당 터널링 콘솔 창을 닫아 프로세스를 종료합니다.
- **모든 터널링 일괄 종료**: **`stop_tunnel.bat`** 파일을 더블클릭하여 실행하면 백그라운드에 열려 있는 모든 `ssh.exe` 프로세스가 일괄로 안전하게 정리됩니다.
