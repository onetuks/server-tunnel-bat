# SSH 터널링 자동화 도구 사용 방법

이 도구는 개발, 품질, 운영 환경의 SSH 터널링 연결과 패스워드 입력을 자동화해 주는 도구입니다.

## 1. 사전 설정 (tunnel_config.json)
실행하기 전에 `tunnel_config.json` 파일을 열어 원하는 환경의 접속 정보를 설정해 주어야 합니다.
- Enabled: 해당 터널링을 실행할지 여부 (true: 실행, false: 실행 안 함)
  * 여러 개의 환경을 true로 설정하면, 더블클릭 한 번으로 여러 개의 터널링을 동시에 띄울 수 있습니다.
  * 여러 개를 동시에 띄울 때는 로컬 포트(LocalPort1, LocalPort2)가 서로 중복되지 않도록 주의해야 합니다.
- RemoteHost / JumpHost: 각 환경의 SSH 접속 계정 및 IP 정보
- RemotePassword / JumpPassword: 각 서버에 로그인할 때 자동으로 입력될 비밀번호

## 2. 터널링 실행 (run_tunnel.bat)
1. `run_tunnel.bat` 파일을 실행합니다. (더블클릭)
2. `tunnel_config.json`에서 Enabled가 true인 환경의 터널링 창(classic console)이 순차적으로 실행됩니다.
3. 터널링 창이 생성된 후 비밀번호가 자동으로 입력될 때까지 마우스 클릭이나 키보드 조작을 하지 않고 잠시 대기합니다. (클릭을 하면 창 포커스가 풀려 비밀번호 자동 입력이 실패할 수 있습니다.)

## 3. 연결 확인 방법
- 생성된 터널링 창에 최종 서버의 쉘 프롬프트(예: [cesadmin@... ~]$)가 출력되었는지 확인합니다.
- 웹 브라우저를 열어 설정한 로컬 포트(예: http://localhost:8090 등)를 통해 서비스 페이지가 정상 호출되는지 확인합니다.

## 4. 터널링 종료 (stop_tunnel.bat)
- 특정 터널링만 종료하려면, 해당 터널링 콘솔 창을 닫아 종료합니다.
- 전체 터널링을 한 번에 일괄 종료하려면 `stop_tunnel.bat` 파일을 실행합니다. (더블클릭 시 백그라운드의 모든 ssh 프로세스가 종료됩니다.)
